CORE CAPABILITIES

Built for Agentic Scale

Upend legacy limitations. Agentic commerce needs systems designed for sub-cent, machine-driven payments.

ZERO GAS

Gas-Free Transfers

Agents sign Ed25519 payment authorizations offchain — no gas, no onchain transaction per payment. The TEE aggregator verifies signatures, deducts balances, and confirms to sellers instantly. The cost of a transaction never exceeds its value.

PHALA TEE

TEE Verifiable Execution

The aggregator runs inside a Phala SGX enclave — tamper-proof, remotely attestable. Every balance deduction and batch settlement is cryptographically verifiable. No trust assumptions beyond the hardware.

Batched Settlement

Thousands of micropayments are netted and settled in a single onchain transaction. Bilateral netting minimizes Jetton transfers.

Spending Policies

Per-agent spending limits, daily caps, and configurable thresholds. Fine-grained control over autonomous agent spending.

Human-in-the-Loop

Large payments trigger Telegram approval requests. Humans stay in control while agents operate autonomously for routine transactions.

x402 Compatible

HTTP 402 Payment Required protocol. Sellers return payment requirements, buyers sign and retry — seamless machine-to-machine commerce.

PAYMENT FLOW

How It Works

01

Deposit USDT

Agent deposits USDT to the CyberGateway smart contract on TON. One-time onchain transaction.

02

Sign Offchain

For each payment, agent signs an Ed25519 authorization offchain. Zero gas, instant — like signing a check.

03

TEE Verifies

Phala TEE aggregator verifies the signature, checks balance, deducts, and confirms to the seller — all inside a secure enclave.

04

Batch Settle

TEE periodically submits batchSettle() to the CyberGateway contract. Thousands of payments, one onchain tx.

DEVELOPER EXPERIENCE

Integrate in Minutes

SDK for buyers, HTTP gateway for sellers. x402 protocol out of the box.

Buyer Agent — SDK 
import { NanoPayClient } from "@cyberpay/nanopay";

const client = new NanoPayClient({
  keypair,
  teeUrl: "https://tee.cyberpay.org"
});

// Pay $0.001 for an API call — zero gas
const receipt = await client.pay({
  to: "seller-address",
  amount: 1000,  // 0.001 USDT
});
// ✅ Instant confirmation, TEE-signed receipt

Sign → TEE verify → Instant receipt. No onchain tx needed.

Seller — x402 Gateway 
import { requirePayment } from "@cyberpay/gateway";

// Gate any route behind nanopayment
app.get("/premium-data",
  requirePayment({
    amount: "1000",  // 0.001 USDT
    to: "your-ton-address"
  }),
  (c) => c.json({ data: "premium content" })
);
// No payment? → 402 with requirements
// Has payment? → TEE verify → serve resource

HTTP 402 protocol. One middleware, instant monetization.

TRUST MODEL

Why TEE Matters

Traditional payment channels require onchain state for every update. TEE eliminates that overhead while preserving verifiability.

Tamper-Proof Execution

Code runs inside Intel SGX enclaves on Phala Network. Even the server operator cannot read or modify the aggregator's state. Balance tracking is cryptographically sealed.

Remote Attestation

Anyone can verify the exact code running inside the enclave via attestation reports. The TEE public key, platform info, and code hash are all publicly auditable.

Verifiable Receipts

Every payment produces a TEE-signed receipt with Merkle proof inclusion. Receipts are independently verifiable without trusting any third party.
